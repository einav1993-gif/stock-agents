## מה חדש בגרסה 2

**1. שכבת נתונים עמידה (`data_layer.py`)**
Yahoo Finance חוסמת שרתים של GitHub Actions — זו הסיבה שהדוחות היו ריקים
(RSI ריק, "טרנד לא ידוע", ציונים של 1–11). עכשיו כל משיכת נתונים עוברת דרך
שרשרת גיבויים: yfinance → Stooq. אם שניהם נכשלים — המערכת *אומרת* את זה.

**2. ראש צוות שבאמת מבקר (`team_lead.py`)**
סופר כמה מניות קיבלו נתונים אמיתיים מכל סוכן. כיסוי מתחת ל-50% ⇒ הדוח
מסומן באדום "לא אמין — לא לסחור", בטלגרם ובאתר. מניה בלי נתונים מקבלת
"⚫ אין נתונים" במקום המלצה מזויפת.

**3. לולאת למידה אמיתית (`self_review.py` + `end_of_day.yml`)**
- ההמלצות נשמרות ב-`data/tracking.json` **שמקומט לריפו** — ההיסטוריה שורדת בין ריצות
  (קודם הקובץ נכתב לתיקייה שנמחקה בסוף כל ריצה, אז שום דבר לא נשמר).
- הבדיקה נעשית על **הגבוה/נמוך של היום** — לא רק על הסגירה
  (קודם מניה שנגעה ביעד באמצע היום ונסגרה נמוך נחשבה בטעות "החטאה").
- כל סוכן מקבל ציון דיוק אישי: הצביע למעלה והמניה עלתה? נקודה. טעה? מינוס.
- המשקלות ב-`data/weights.json` מתעדכנים בהדרגה: סוכן מדויק ⇒ משקל גבוה יותר מחר.
  (עדכון איטי בכוונה — יום חריג אחד לא הופך את המערכת.)
- הלקחים נכתבים ל-`data/learning_log.json` ומוצגים באתר ובטלגרם.

**4. תיקוני באגים**
קריסה כש-VIX לא נטען, עמודות MultiIndex של yfinance החדש, import של pandas
בתחתית קובץ, כפילות רישום בהרצה ידנית חוזרת.

## הפעלה

הכול אוטומטי דרך GitHub Actions (שני ה-workflows בתיקייה `.github/workflows`).
להרצה ידנית: Actions → בחרי workflow → Run workflow.

Secrets נדרשים (Settings → Secrets → Actions):
`TELEGRAM_TOKEN`, `TELEGRAM_CHAT_ID`, ואופציונלי `FINNHUB_API_KEY`.

בדיקה מקומית: `python3 test_system.py`
